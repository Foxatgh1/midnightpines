# Reflection Paragraph (strengths and weaknesses of vibe coding)

A major strength of vibe coding was speed and iteration: I could describe exact enemy behaviors in plain language and quickly test whether the generated code matched my design. It was especially useful for implementing different AI movement patterns and tuning gameplay balance. A weakness is that AI output still needs careful verification, because details like speed ordering, win/loss state transitions, and collision logic can be subtly wrong without deliberate testing. I learned that vibe coding works best when I define precise constraints, test frequently, and then refine prompts based on observed behavior.
